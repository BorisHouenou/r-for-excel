6.7 group_by() %>% summarize()

In R, we can create the functionality of pivot tables by using 2 main dplyr verbs in combination: group_by and summarize.

Say it with me: âpivot tables are group_by and then summarizeâ. And just like pivot tables, you have flexibility with how you are going to summarize. For example, we can calculate an average, or a total.

I think itâs incredibly powerful to visualize what we are talking about with our data when do do these kinds of operations. It looks like this (from RStudioâs cheatsheet; all cheatsheets available from https://rstudio.com/resources/cheatsheets):

Letâs do this in R.

6.7.1 group_by one variable

Letâs try this on our lobsters data, just like we did in Excel. We will count the the total number of lobster by year. In R vocabulary, we will group_by year and then summarize by counting using n(), which is a function from dplyr. n() counts the number of times an observation shows up, and since this is uncounted data, this will count each row. Weâll also use the pipe operator %>%, which you can read as âand thenâ.

6.8.2 dplyr::count()

Now that weâve spent time with group_by %>% summarize, there is a shortcut if you only want to summarize by count. This is with a function called count(), and it will group_by your selected variable, count, and then also ungroup. It looks like this:

lobsters %>%
  count(site, year) 

## This is the same as:
lobsters %>%
  group_by(site, year) %>% 
  summarize(n = n()) %>%
  ungroup()
